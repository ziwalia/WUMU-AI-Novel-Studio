## 📖 项目简介

乌木智书是一款 AI 驱动的长篇小说创作工具，帮助作者从构思到成稿完成一整部小说的创作。通过智能化的向导流程和全自动生成能力，你可以专注于创意和故事本身，让 AI 协助你完成从世界观构建到大结局定稿的全过程。

## ✨ 功能特性

### 🎯 智能创作向导

从零开始，逐步引导完成一部完整的小说创作：

```
项目信息 → 小说架构 → 小说大纲 → 章节目录 → 草稿生成 → 一致性审校 → 改写 → 定稿 → 导出
```

每一步骤的功能：

| 步骤 | 功能说明 |
|------|---------|
| **项目信息** | 填写小说名称、类型、章节数、字数目标、故事梗概、叙事视角、创作指导等 |
| **小说架构** | AI 生成核心使命、世界观设定、主线情节、角色体系、角色关系、叙事风格 |
| **小说大纲** | AI 按阶段划分，生成每阶段的主题、章节范围、关键事件、情感基调和角色变化 |
| **章节目录** | AI 为每一章生成标题和概要，自动检测并去重雷同章节 |
| **草稿生成** | AI 根据章节目录和上下文生成每章初稿 |
| **一致性审校** | AI 从 11 个维度审校章节质量并评分 |
| **改写** | AI 根据审校意见针对性改写 |
| **定稿** | 确认最终文本，标记章节完成 |
| **导出** | 支持 TXT / Markdown / JSON 三种格式导出 |

### 🤖 AI 全自动生成

设置好小说名称和类型后，一键启动全自动生成：

- **端到端自动**：自动完成 架构 → 大纲 → 目录 → 全部章节草稿 → 审校 → 改写 → 定稿
- **可配置审校轮数**：支持 1-5 轮审校改写，每轮先审校评分再针对性改写
- **智能跳过**：审校评分 ≥ 90 分自动跳过改写，节省 token 消耗
- **章节去重**：自动检测章节目录中重复或雷同的标题和情节，最多 5 轮自动重写
- **断点续传**：中断后可从上次完成的章节继续生成
- **全文审核**：全部章节完成后，自动触发 4 轮全文审核并评分

### 👥 角色追踪系统

AI 自动追踪和管理角色状态，确保全书记忆一致：

- **角色属性**：姓名、权重（主角/重要配角/配角/龙套）、年龄、性格、能力、基础信息
- **角色状态追踪**：自动追踪生存状态（存活/死亡）、当前位置轨迹、情绪状态、对话关键词
- **角色成长弧线**：记录每个角色的成长变化历史
- **能力追踪**：自动记录角色获得的新能力、持有的道具
- **角色快照**：每章保存角色状态快照，改写时可回滚

### 🔗 角色关系网络

- **8 种关系类型**：恋人、师徒、敌对、同门、盟友、朋友、亲人、其他
- **关系可视化**：彩色标识不同关系类型
- **动态追踪**：AI 自动检测角色间新增关系和关系变化
- **关系描述**：每段关系附带具体描述

### 🪝 伏笔管理系统

AI 自动埋设和收束伏笔，确保叙事完整性：

- **5 种伏笔类型**：主线伏笔（MF）、动作伏笔（AF）、角色伏笔（CF）、设定伏笔（SF）、预言伏笔（YF）
- **自动埋设**：AI 写作时自动埋入伏笔并记录
- **自动收束**：AI 检测到伏笔回收时机时自动收束
- **伏笔提醒**：生成最后 2 章时自动提醒 AI 收束所有未完成的伏笔
- **优先级标记**：高/中/低三级优先级

### 📊 章节元数据提取

每章生成后，AI 自动提取结构化元数据：

- **章节摘要**：50-100 字概括核心事件
- **时间线**：故事内的时间节点
- **场景类型**：战斗升级、修炼突破、感情升温、日常闲聊等 20+ 种场景分类
- **节奏标签**：紧张（tension）/ 舒缓（calm）/ 过渡（transition）
- **情绪强度**：高/中/低
- **角色变化**：新角色登场、状态变化、位置移动、能力获取、生死变化
- **关系变化**：新增关系、关系类型变化

### 📝 一致性审校系统

AI 从 11 个维度审校每一章：

1. **章节标题**：格式正确性、与目录一致性
2. **章节开头**：场景切入是否自然完整
3. **角色一致性**：行为、性格、说话风格是否与设定一致，位置是否合理，已死亡角色是否不该再出场
4. **情节连贯性**：逻辑漏洞和前后矛盾
5. **时间线**：时间推进是否合理
6. **伏笔管理**：遗忘的伏笔需要回收，可埋设新伏笔
7. **文笔质量**：重复用词、生硬表达
8. **前后连贯性**：与前几章的剧情逻辑是否一致，设定冲突或时间线错误
9. **场景多样性**：本章场景类型是否与前几章过度重复
10. **过渡段质量**：开头是否与上一章结尾形成自然衔接
11. **计划完成度**：是否完成了章节目录中计划的核心事件

审校评分机制：
- **≥ 90 分**：质量优秀，跳过改写
- **80-89 分**：存在可改进之处，执行改写
- **< 80 分**：质量问题较多，执行改写

### 🔄 改写系统

- AI 根据审校意见提取具体问题列表
- 针对性修复问题，保留原文整体结构和节奏
- 不大幅改动没有问题的部分
- 支持严格字数控制模式（±5%）
- 改写后重新提取章节元数据

### 📖 4 轮全文审核

全部章节定稿后，自动执行 4 轮全文审核：

| 轮次 | 审核重点 | 维度数 |
|------|---------|--------|
| **Pass 1 - 结构总览** | 整体架构、主线情节、情节连贯性、节奏控制、角色成长弧线、伏笔管理、段落衔接、高潮设计、结局满意度、主题深度 | 10 |
| **Pass 2 - 开篇审核** | 开篇吸引力、世界观构建、主角塑造、文笔风格、对话质量、场景描写、情感渲染、读者代入感 | 8 |
| **Pass 3 - 中段采样** | 冲突升级、配角塑造、伏笔运用、节奏变化、情感张力、世界观扩展、叙事视角一致性 | 7 |
| **Pass 4 - 结局审核** | 高潮爆发力、伏笔回收率、角色结局合理性、主题升华、情感余韵、结局完整性 | 6 |

最终生成综合评分、各维度评分、总体评价和具体整改建议。

### 🧠 上下文管理系统

草稿生成时，每章自动注入丰富的结构化上下文，确保 AI 写作连贯：

- **前 1-2 章完整内容**：确保场景衔接
- **前几章结尾段落**（各 500 字）：延续悬念
- **递进总摘要**：全书进展概览
- **近 10 章摘要**：近期剧情回顾
- **近 5 章场景类型/节奏/情绪**：避免场景重复
- **活跃角色状态**：主角 + 近 3 章出场角色的位置、情绪
- **未收束伏笔清单**：提醒 AI 收束时机
- **当前时间线**：确保时间推进合理
- **用户剧情指引**：支持用户预设下一章走向
- **完整角色表 + 关系网**：每章都注入全量角色信息
- **完整章节目录**：每章都注入全量目录信息

### 📤 多格式导出

## 📚 使用指南

### 1. 配置 AI 模型

首次使用需要配置 LLM 接口：

1. 点击右上角 ⚙️ **设置** 按钮
2. 点击 **新增配置**
3. 选择 LLM 接口格式（如 OpenAI、Ollama、LM Studio 等）
4. 填入 API 地址和密钥
5. 选择模型
6. 点击 **测试连接** 确认可用
7. 可创建多个配置，随时切换

### 2. 创建新项目

1. 点击左侧 **新建项目**
2. 输入项目名称
3. 点击创建，进入创作向导

### 3. 填写项目信息

在 **项目信息** 步骤中填写：

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| 小说名称 | 作品标题 | ✅ |
| 小说类型 | 从 209 种内置类型中选择 | ✅ |
| 总章数 | 计划生成的章节数 | ✅ |
| 每章字数 | 每章目标字数 | ✅ |
| 严格字数控制 | 开启后强制 ±5% 字数范围 | 选填 |
| 故事梗概 | 故事的大致走向 | 选填 |
| 叙事视角 | 第一人称 / 第三人称有限 / 第三人称全知 | 选填 |
| 创作指导 | 对 AI 的额外创作要求 | 选填 |
| 文笔风格 | 从 13 种预设风格中选择 | 选填 |
| 核心角色 | 主要角色描述 | 选填 |
| 关键道具 | 故事中的重要道具 | 选填 |
| 场景地点 | 主要场景设定 | 选填 |
| 时间压力 | 故事内的时间紧迫感 | 选填 |

### 4. 手动创作流程

按左侧步骤栏依次进行：

| 步骤 | 说明 |
|------|------|
| **项目信息** | 填写小说基本参数，设置创作方向 |
| **小说架构** | AI 生成世界观、角色体系、主线情节、关系网络（JSON 格式，卡片展示） |
| **小说大纲** | AI 按阶段生成大纲，包含关键事件和角色变化 |
| **章节目录** | AI 为每章生成标题和概要，自动去重检测 |
| **草稿生成** | AI 根据目录和上下文生成每章初稿，支持流式输出 |
| **一致性审校** | AI 从 11 个维度审校并评分，输出详细审校意见 |
| **改写** | AI 根据审校意见针对性改写问题部分 |
| **定稿** | 确认最终文本，标记章节完成 |
| **导出** | 选择格式导出 TXT / Markdown / JSON |

### 5. 全自动生成（推荐）

填写完小说名称和类型后，点击项目信息页底部的 **全自动生成** 按钮：

1. 设置审校改写轮数（1-5 轮，默认 1 轮）
2. 点击开始确认
3. 系统自动完成：
   - 生成小说架构 → 提取角色和关系
   - 生成小说大纲
   - 生成章节目录 → 自动去重检测和重写
   - 逐章循环：草稿 → 审校评分 → 改写 → 定稿
   - 全部完成后执行 4 轮全文审核
4. 随时可点击中止，下次可从断点继续

## 💻 开发

### 环境要求

- [Node.js](https://nodejs.org/) >= 18
- [Rust](https://www.rust-lang.org/tools/install) (桌面版开发)
- [Tauri CLI](https://tauri.app/guides/prerequisites/)

### 本地运行

```bash
# 克隆仓库
git clone https://github.com/ziwalia/WUMU-AI-Novel-Studio.git
cd WUMU-AI-Novel-Studio

# 安装依赖
npm install

# 网页版开发模式
npm run dev

# 桌面版开发模式
npm run tauri dev
```

### 构建

```bash
# 构建前端
npm run build

# 构建桌面安装包
npm run tauri build
```

- 流式输出支持 `stream_options.include_usage`
- Token 用量实时统计

**类型系统**
- 内置 209 种网络小说类型/流派（男频 + 女频）
- 内置 13 种文笔风格预设
- 类型和风格数据自动注入创作 prompt

**UI 改进**
- 主题模式：亮色 / 暗色 / 跟随系统
- 全局字体配置（字体、字号、行高）
- Toast 通知系统
- 侧边栏可折叠、宽度可调
- 右侧面板：上下文/角色双标签页
- 状态栏显示生成进度和 Token 统计
